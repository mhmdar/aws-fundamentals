  • Most cloud VMs, containers & serverless runtimes are Linux under the hood
  • DevOps stack assumes Linux: Docker, Kubernetes, Jenkins, Terraform, Ansible
  • AWS default AMIs, SSH access & cloud-init scripts are Linux-first

  • Certifications & labs — AWS, LFCS, and today's EC2 lab all need CLI comfort

Linux CLI cheat sheet

📁 Navigate

  • pwd print working directory
  • cd dir change directory
  • cd .. go up one level
  • cd ~ go home
  • cd - previous directory
  • ls -la list all (+ hidden)
  • ls -lh human-readable sizes

📄 View & edit

  • cat file view entire file
  • less file page through file
  • head -n 20 file first N lines
  • tail -f log follow live log
  • nano file simple editor
  • vi file power editor (Esc :wq)
  • touch file create / update timestamp

✏️ Manage files

  • cp src dest copy file
  • cp -r dir dest copy folder
  • mv src dest move / rename
  • rm file delete file
  • rm -rf dir delete folder (careful!)
  • mkdir -p a/b/c create nested dirs
  • ln -s target link symbolic link

🔍 Search & filter

  • grep "text" file search in file
  • grep -r "text" . search recursively
  • grep -i "text" file case-insensitive
  • find . -name "*.log" find by name
  • find . -type f -size +100M large files
  • which cmd path to command
  • history | grep ssh search history

🌐 Network & SSH

  • ssh -i key.pem user@IP connect to EC2
  • scp -i key file user@IP: copy file over SSH
  • curl http://IP test HTTP response
  • curl -I URL headers only
  • ping -c 4 host test reachability
  • ss -tlnp open TCP ports
  • wget URL download file

⚙️ System & permissions

  • sudo cmd run as admin
  • systemctl start nginx start a service
  • systemctl status svc check service
  • systemctl enable svc start on boot
  • chmod 400 key.pem lock down key file
  • chmod +x script.sh make executable
  • chown user:group file change owner

📊 Disk & memory

  • df -h disk space summary
  • du -sh * folder sizes here
  • du -sh dir size of one folder
  • free -h RAM & swap usage
  • lsblk block devices & mounts
  • mount mounted filesystems
  • fdisk -l list partitions (sudo)

🔄 Processes

  • ps aux all running processes
  • top live process monitor
  • ps aux | grep nginx find a process
  • kill PID graceful stop
  • kill -9 PID force kill
  • jobs / bg / fg background jobs
  • nohup cmd & survive logout

🔗 Pipes, help & misc

  • cmd > file redirect to file

Amazon S3 — What is it? HANDS-ON

Amazon S3 (Simple Storage Service) is AWS's managed object storage service — store and retrieve any amount of data from anywhere on the web.

Object storage treats data as discrete objects (file + metadata), not blocks on a disk or rows in a table. Each object lives in a bucket and is accessed by a unique key (its path/name).

How object storage works

  • Upload a file → stored as an object
  • Flat namespace — keys like photos/2025/cat.jpg
  • HTTP/HTTPS API access — Console, CLI, SDKs
  • 99.999999999% (11 nines) durability

Not a filesystem

  • No true folders — / in keys is a naming convention
  • Not mounted like C:\ or /home
  • Optimized for scale, durability & cheap storage
  • Ideal for files you read/write whole — not databases

Amazon S3 — vs local & network storage

Local disk Network storage (NAS / file server) Amazon S3
Where Inside your PC or EC2 instance Shared drive on office LAN AWS cloud — region you choose
Access One machine only Mount as network drive (\\server\share) API / Console / CLI from anywhere
Scale Fixed disk size (256 GB, 1 TB…) Limited by NAS hardware Virtually unlimited objects
Durability Disk failure can lose data RAID helps; still one location Replicated across multiple facilities
Cost Buy hardware upfront Buy NAS + maintain it Pay per GB stored + requests
Best for OS, apps, temp processing Office documents, team shares Backups, web assets, data lakes, archives

Rule of thumb: use local/EBS for running apps; use S3 for files you want durable, shared, and internet-accessible.

Amazon S3 — key terminology

BucketContainer for objects. Name is globally unique across all AWS accounts.
ObjectA file + metadata. Can be 0 bytes up to 5 TB in size.
KeyFull path/name of an object, e.g. images/logo.png. Looks like a folder path but isn't a real directory.
RegionGeographic location where the bucket lives (e.g. eu-central-1). Choose close to users.
VersioningKeep multiple versions of an object — recover from accidental overwrites or deletes.
Storage classCost/retrieval tier — Standard, Infrequent Access (IA), Glacier for archives.
Block Public AccessAccount/bucket setting that prevents accidental public exposure (on by default).
ARNAmazon Resource Name — unique ID used in IAM policies, e.g. arn:aws:s3:::my-bucket/*

Amazon EC2 — virtual servers & networking

What is EC2?

Elastic Compute Cloud — AWS's core IaaS service. Rent virtual machines (servers) on demand; scale up or down in minutes.

  • Instance — one running VM built from an AMI + instance type; has CPU, RAM, disk & a network interface
  • AMI — OS image template (today: Amazon Linux 2023)
  • Instance type — hardware profile; lab uses t3.micro (2 vCPU, 1 GiB RAM)
  • Key pair — .pem file for SSH; EBS — persistent disk attached to the instance

Where instances live (VPC)

  • CIDR — IP range notation; VPC 10.0.0.0/16 = 65,536 private addresses
  • Availability Zone (AZ) — isolated datacenter; subnets exist in exactly one AZ
  • Subnet — slice of VPC CIDR (e.g. 10.0.1.0/24); public subnets route to Internet Gateway
  • Security group — stateful virtual firewall on the instance; controls inbound/outbound traffic

Today's lab: create your own VPC + public subnet, launch EC2 with public IP for SSH & HTTP. Production often adds private subnets + ALB.

Connect (SSH)

Mac/Linux:

chmod 400 ~/.ssh/aws-club-xx-key.pem
ssh -i ~/.ssh/aws-club-xx-key.pem ec2-user@PUBLIC_IP

Windows (PowerShell):

icacls "$env:USERPROFILE\Downloads\aws-club-xx-key.pem" /inheritance:r
icacls "$env:USERPROFILE\Downloads\aws-club-xx-key.pem" /grant:r "$env:USERNAME:R"
ssh -i $env:USERPROFILE\Downloads\aws-club-xx-key.pem ec2-user@PUBLIC_IP

Type yes on first connect. Prompt: [ec2-user@ip-10-0-1-xx ~]$

Deploy nginx + welcome page

On EC2:

sudo dnf install -y nginx
sudo systemctl enable --now nginx
sudo tee /etc/nginx/default.d/aws-club-lab.conf <<'EOF'
location / {
    gzip off;
    sub_filter_types text/html;
    sub_filter '__PRIVATE_IP__' '$server_addr';
    sub_filter '__HOSTNAME__' '$hostname';
    sub_filter_once off;
}
EOF
sudo nginx -t && sudo systemctl reload nginx
curl -s http://localhost | grep ip-10

On laptop (from materials/):

scp -i KEY.pem lab-assets/lab-b-ec2/index.html ec2-user@PUBLIC_IP:~/

On EC2:

sudo cp ~/index.html /usr/share/nginx/html/index.html
